# VProtocol MiniGame Guide

## Цель
Разработка встраиваемой 2D математической mini-game в жанре Tower Defence: робот защищается от вирусов, нанося урон за правильные ответы на примеры.

## Продуктовое видение
- Core loop: `Враг идет к роботу -> игрок решает пример -> робот стреляет -> враг получает урон`.
- Ошибка в ответе снимает 1 слой защиты робота.
- Проигрыш: все слои защиты пробиты, робот взломан.
- Победа: выдержать волну (позже несколько волн).

## Ограничения MVP
- Возрастная рамка: 6-12.
- Операции: `+`, `-`, `*`, `/` (деление только без остатка).
- Ответы: 3 варианта, 1 правильный.
- Ввод: touch-first (Input System), поддержка клика в Editor.
- Без resistance/иммунитетов врагов, без мета-прогрессии.

## Боевые и математические правила
- Базовый урон зависит от правильного результата примера.
- Комбо: при серии быстрых правильных ответов (порог по времени) повышается сложность следующего вопроса и растет множитель урона.
- Множитель только положительный (награда, не штраф).
- Диапазоны чисел и доступные операции задаются конфигом уровня.

## Кандидаты типов врагов (старт)
- `SpamBot`: быстрый, малый HP, давит количеством.
- `BruteWorm`: медленный, высокий HP, давит прочностью.
- `Phisher`: средний HP/скорость, вариант для расширения (особое поведение в следующих итерациях).

## Правило разработки
Работа ведется крупными блоками фич. После каждого блока обязателен checkpoint: оценка модульности, выявление связности, точечный рефакторинг до перехода к следующему блоку.
